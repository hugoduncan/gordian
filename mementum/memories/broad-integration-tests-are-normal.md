💡 In `gordian tests`, broad structural footprint is descriptive, not automatically bad. Integration-style tests legitimately have high reach / broader closures; the smell is a unit-style test that becomes integration-ish without signaling intent. This led to the rule: classify tests first (`:unit-ish` / `:integration-ish` / `:mixed`), then only flag `unit-test-too-broad` conservatively (breadth threshold + no integration cue), while treating broad integration tests as low-severity or informational. This avoids punishing healthy wiring tests.
